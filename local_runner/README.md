# 로컬 자동 일정 수집기 (local_runner)

PC에서 실행하는 독립 도구입니다. 기존 스크립트/GitHub 워크플로우와 분리되어 있어 영향을 주지 않습니다.

## 실행
- `run.bat` 더블클릭 → 로컬 서버가 뜨고 브라우저에 제어 페이지가 열립니다.
- 또는 터미널에서: `python local_runner/u18_auto.py`

## 동작
- 시작 즉시 **1회 수집**(오늘 경기가 있을 때) → 이후 **30분마다** 갱신.
- 자동 수집 조건: **오늘(KST) 경기가 있고 09:00~21:00** 일 때. (그 외에는 건너뜀)
- 수집 = 저장소 루트의 `build_all.py --incremental` (일정 증분 + docs 반영).
- 수집 후 변경된 일정 파일을 **git commit & push** 로 GitHub에 반영합니다.
  - 대상: `u18_schedule.json`, `u18_schedule.html`, `u18_schedule_data.js`, `docs/u18_schedule.html`, `docs/u18_schedule_data.js`
  - push 전에 `git pull --rebase` 로 원격(액션 커밋 등)과 충돌을 방지합니다.

## 제어 페이지
- 오늘 경기 수 / 마지막 수집·결과 / 다음 예정 / 로그 표시.
- **[지금 수집]** 즉시 1회 실행, **[종료]** 서버 종료.
- 브라우저를 닫아도 종료 전까지 백그라운드로 계속 동작합니다.

## 주의
- 이 PC에 git 인증(원격 push 권한)이 설정돼 있어야 합니다.
- GitHub의 일정 증분 워크플로우(매일 12/18/21시)와 동시에 쓰면 양쪽이 push 하므로,
  로컬로 운영하는 날은 워크플로우를 수동만 쓰거나 비활성화하는 것을 권장합니다(필수는 아님 — rebase로 합쳐짐).
